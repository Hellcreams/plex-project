# plex-project
### 1. 서버 사양의 선택 이유
#### 아버지의 요구사항
* 가정에 드는 전기비에 매우 민감하기 때문에, 전력을 최소로 하는 기기
* 기기 소음에 의해 민감하기에 저소음으로 돌아갈 수 있는 기기
#### 서비스의 요구사항
* 동작 서비스 : plex, prometheus, grafana, 자체 제작 exporter, node-exporter, (photoview)
* *Ubuntu Server* 기반 동작
* subtitle brun-in 현상을 제거 후 운영
* 영상 스트리밍 서비스는 큰 비용을 요구하지 않는 것을 반영
* 불필요한 트래픽 방지를 위한 외부 인터넷망 연결 차단
* DVD 리핑, 변환 후 용량 예측 및 백업 하드 (RAID는 필요성을 못 느낌)
#### 기타
* 구매 비용은 넉넉해서, 고려 대상이 후순위였음.
* 중고 기기를 사는 선택지도 있으나, 요구사항에 비적합, 불필요한 추가 비용 발생 가능성을 배제하기 위해 신품으로 구매함.

### 2. python-exporter
#### 왜 만들었는가?
* [[#1. 서버 사양 의 선택 이유]|요구 조건]]에 따라, [[#DVD 영상|자료]]를 잘못 바꿀 경우 기기에 치명적인 무리가 가는 subtitle burn-in 현상이 발생할 수 있는 조건이였음.
* plex 자체에는, 제공되는 영상 서비스마다 그대로 스트리밍, 트랜스코드가 발생하는 등을 알려주는 api가 존재했음.
* 이에 subtitle burn-in 현상이 발생하는지 실시간으로 모니터링 할 수 있는 체계의 필요성을 느꼈음.
#### 무엇을 고려했는가?
* 프로그램 구현을 위해 가장 자신있는 언어가 python 이였고, 따라서 python 사용하기로 결정.
* photoview 설치 시 겪었던 과도한 의존성 문제를 해결하기 위해, 오직 python의 **내장 모듈만**을 사용하여 구축 시 시간이 최소화하도록 계획함.
* plex api로부터 정보를 수집하고, 동시에 prometheus 데이터를 제공.
	* thread를 통해, 동시성 문제 해결
	* prometheus의 파이프라인에 대한 이해
* systemd 서비스를 직접 등록
* /opt에 경로 설정, 권한 설정 후 관리
