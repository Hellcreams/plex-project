# Plex-Project
## 1. 시작 배경
* 아버지께서 오랫동안 가지고 계셨던  DVD 영화를 70여장 정도 가지고 계셨습니다.
* 하지만 시대가 변화하며 가정에서 DVD보다는 OTT 서비스를 통한 TV 시청으로 생활상이 변화하였고, 점차 가지고 있던 DVD는 활용되지 않은 채로 보관되고 있었습니다.
* 이에 기존 DVD를 디지털화하여 Plex 기반의 로컬 OTT 서비스를 구축하고, 스마트 TV와 연계하여 DVD 삽입 또는 교체할 필요 없이 리모콘만으로 편하게 시청할 수 있는 시스템을 만들기로 결심하였습니다.

## 2. 프로젝트 목표
1. 저작권법(제 30조 등)을 준수하는 범위 내에서 DVD를 리핑하여 개인 영상물로 저장합니다.
2. 가정 및 소프트웨어의 요구사항을 통합적으로 고려하여 적합한 하드웨어(컴퓨터) 구매하고 인프라를 구축합니다.
3. 안정적인 운영을 위하여 모니터링 대상을 정의하고 그에 맞는 모니터링 체계를 구현합니다.
4. 삼성 스마트 TV와의 연계를 통해 기존 DVD 영상 시청에 따른 불편함을 제거합니다.
5. 추후 기기가 고장하거나 교체될 것을 대비하여, 쉬운 재배포 (docker-compose) 및 클라우드 이식 방안을 마련합니다. *(예정)*

## 3. 구현 내용
### 1) DVD 리핑 및 인코딩
#### DVD 데이터 변환 정보
##### a. 공통
* 해상도 : 480p, 30fps, RF17
* 인코딩 : MPEG-2 -> H.264
* 사운드 : AC3 Passthrough
* 저장 대상 : 본편, 보너스 영상 (타 영화 예고편은 제외)
* 주요 고려사항 : 원본 유지(1), 용량 감소(2)
##### b. plex 재생용
* 컨테이너 : MP4
* 저장 매체 : 본체 SSD (main), HDD (backup)
* 트랜스코딩(subtitle burn-in) 방지를 위한 자막판, 더빙판 2편으로 분기
* 주요 고려사항 : 하드웨어에 최적화된 인코딩
##### c. 원본 저장용
* 컨테이너 : MKV
* 저장 매체 : Blue-ray Disk
* 주요 고려사항 : 장기 보관

#### DVD 데이터 변환 과정
* *DVD_decrypter.exe*를 사용하여, 각 DVD를 리핑하여 별도 SSD에 저장한다.
* MakeMKV를 통해, 필요한 영상만을 추출하여 [[#DVD 데이터 변환 정보|소스 정보]]에 맞춰 인코딩한다.

### 2) 요구사항에 따른 하드웨어 자원 선정
#### 최종안
* CPU : Intel N150
* RAM : 8GB (노트북용)
* SSD : 512GB
* HDD : 1TB
#### 고려사항
##### 요구사항
1. 부모님의 전기비 및 기기 소음에 대한 우려를 반영하여 CPU를 선정하였습니다.
2. 기기 내 작동하는 OS가 *Ubuntu Server* 이고, 서비스가 Plex, Prometheus, Grafana, Node-Exporter, [자체 제작 프로그램]인 점을 고려하여 적절한 CPU 및 RAM 용량을 고려하였습니다.
3.  백업은 진행하되, 하드웨어 구조 상 RAID 구성은 과하다고 판단하여 적절한 SSD, HDD를 선정하였습니다.
4. subtitle burn-in 현상이 발생 시 과도한 CPU 사용 현상이 발생하는 점을 고려하여, 하드웨어 성능 향상 대신 burn-in 현상 발생 요소 제거 및 관련 현상 모니터링 체계를 구축하기로 결정하였습니다.

### 3) Plex 서비스 구축
* *Ubuntu Server* 를 설치하고, 자동 업데이트 방지를 위해 외부 네트워크와의 연결을 차단하도록 조치하였습니다.
* net-tools 등은 다른 *Ubuntu* PC에서 .deb 파일을 다운로드하고, Plex 등의 프로그램은 .tar 등으로 저장하여, USB로 해당 파일을 옮겨 설치하는 방식으로 진행하였습니다.

### 4) 모니터링 체계
#### 기기 자원 모니터링 체계
* 기존에 알려진 node-exporter, Prometheus, Grafana 연계 방식을 이용하여, 각 프로그램을 설치하고 알려진 메뉴얼에 따라 실행하였습니다.
#### subtitle burn-in 모니터링 체계
* 작업 중 기기에 의도치않게 subtitle burn-in 현상이 일어날 수 가능성을 우려하여, 이를 확인할 수 있는 모니터링 체계가 필요하다고 판단하였습니다.
* Plex 내에서 스트리밍 개체 별 트랜스코딩을 포함한 영상별 상태를 상세히 알려주는 전용 API가 따로 존재했습니다.
* 이를 통해 node-exporter와 Prometheus 간의 데이터 전송 파이프라인을 참고하여, Python 기반 Plex API 연계용 전용 exporter를 제작하였습니다.
* exporter 제작 시, docker 없이 배포를 용이하게 하기 위하여 내부 라이브러리만을 이용해 구현하였습니다. (python-exporter.py 참고)
